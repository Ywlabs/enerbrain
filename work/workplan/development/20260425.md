## 2026-04-25 작업 계획

### 목표

- PAS 인증/권한 흐름을 안정화하고
- 분석항목 CRUD + 실행이력 조회의 첫 기능 사이클을 완성한다.

### 1) 우선순위 작업

1. PAS 권한 세분화 구현
   - `SUPER_ADMIN` 전체 접근
   - 사이트 범위 권한(`TB_USER_SITE_ROLE`) 적용
   - 공통 권한 의존성 함수 정리

2. PAS 핵심 CRUD 1차 구현
   - 사이트 목록/등록 API
   - 프로젝트 목록/등록 API
   - 분석항목 목록/등록 API

3. 분석 실행이력 조회 API
   - `TB_ANALYSIS_RUN` 목록 조회
   - 필터: 분석항목, 상태, 기간

### 2) 기술 작업

- SQLAlchemy 쿼리 리포지토리 구체화
- API 응답 표준(`ApiResponse`) 일관 적용
- 실패 케이스 예외 메시지 정리
- Swagger 예시 요청/응답 추가

### 3) DB/연동 확인

- MariaDB 연결 안정화 확인
- admin 로그인/권한 시나리오 재검증
- 샘플 데이터 기반 CRUD 정상 동작 확인

### 4) 완료 기준 (DoD)

- PAS 로그인 성공 후 보호 라우트 접근 가능
- 권한 없는 사이트 접근 시 403 반환
- 사이트/프로젝트/분석항목 최소 CRUD 1세트 동작
- 실행이력 조회 API에서 `RUN_RSLT_JSON`, `RUN_MODEL_JSON` 확인 가능
