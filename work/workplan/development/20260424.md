# 2026-04-24 개발 계획 (현행화)

## 1) 방향 전환 요약

- 기존 대규모 MLOps 범용 구조에서 오케스트레이션 MVP 중심으로 축소
- 내부 운영(PAS)을 우선 구현하고, 고객사 호출(SAS)은 서비스 API 중심으로 제한

## 2) 확정 기능 범위

1. 사이트 관리
   - 업체명/담당자 등 기본정보 관리
   - 테이블: `TB_SITE`

2. 프로젝트 관리
   - 프로젝트명/담당자/설명 관리
   - 프로젝트 API 키 관리
   - 테이블: `TB_BIZ`, `TB_BIZ_API_KEY`

3. 분석항목 관리
   - 분석항목명, 알고리즘코드, 상태
   - 크론 주기, 실행 파이썬 파일 경로, 실행 함수명, 파라미터 JSON
   - 테이블: `TB_ANALYSIS_ITEM`

4. 서비스 API 관리/테스트
   - 고객사 제공용 API 경로/메서드/설명 관리
   - 테스트 요청/응답 샘플 관리
   - 테이블: `TB_API_SVC`

5. 실행 이력 관리
   - 분석항목별 시작/종료/상태/처리시간/입출력건수/결과JSON/오류
   - 테이블: `TB_ANALYSIS_RUN`

6. 권한/인증
   - PAS(내부): 사용자 로그인(JWT 예정)
   - SAS(고객사): 프로젝트 API 키 필수
   - 전체관리자: `TB_USER.GLOBAL_ROLE_CD='SUPER_ADMIN'`
   - 사이트범위권한: `TB_USER_SITE_ROLE`

## 3) DB 기준

- 기준 파일: `work/workplan/baseline/ener_brain_database_mariadb_v1.sql`
- 이번 정리에서 반영:
  - `TB_USER_AUTH` 분리 제거(인증컬럼을 `TB_USER`에 통합)
  - `TB_BIZ_MBR` 제거
  - 샘플 데이터 추가(`SITE/BIZ/ANALYSIS_ITEM/USER_SITE_ROLE`)

## 4) 다음 구현 우선순위

1. PAS 로그인/JWT + 권한 판정
2. 분석항목 CRUD + 실행기(모듈 경로 실행)
3. SAS 키 검증 + 서비스 API 조회
4. 외부 DB 연동 1종 + 간단 ML 처리 -> `TB_ANALYSIS_RUN` 저장
