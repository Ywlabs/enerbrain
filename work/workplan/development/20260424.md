# 2026-04-24 개발 계획 (현행화)

## 1) 방향 전환 요약

- 기존 대규모 MLOps 범용 구조에서 오케스트레이션 MVP 중심으로 축소
- 내부 운영(PAS)을 우선 구현하고, 고객사 호출(SAS)은 서비스 API 중심으로 제한

## 2) 확정 기능 범위

1. 사이트 관리
   - 업체명/담당자 등 기본정보 관리
   - 테이블: `TB_SITE`

2. 프로젝트 관리
   - 프로젝트명/담당자/설명 관리
   - 프로젝트 API 키 관리
   - 테이블: `TB_BIZ`, `TB_BIZ_API_KEY`

3. 분석항목 관리
   - 분석항목명, 알고리즘코드, 상태
   - 크론 주기, 실행 파이썬 파일 경로, 실행 함수명, 파라미터 JSON
   - 테이블: `TB_ANALYSIS_ITEM`

4. 서비스 API 관리/테스트
   - 고객사 제공용 API 경로/메서드/설명 관리
   - 테스트 요청/응답 샘플 관리
   - 테이블: `TB_API_SVC`

5. 실행 이력 관리
   - 분석항목별 시작/종료/상태/처리시간/입출력건수/결과JSON/오류
   - 테이블: `TB_ANALYSIS_RUN`

6. 권한/인증
   - PAS(내부): 사용자 로그인(JWT 예정)
   - SAS(고객사): 프로젝트 API 키 필수
   - 전체관리자: `TB_USER.GLOBAL_ROLE_CD='SUPER_ADMIN'`
   - 사이트범위권한: `TB_USER_SITE_ROLE`

## 3) DB 기준

- 기준 파일: `work/workplan/baseline/ener_brain_database_mariadb_v1.sql`
- 이번 정리에서 반영:
  - `TB_USER_AUTH` 분리 제거(인증컬럼을 `TB_USER`에 통합)
  - `TB_BIZ_MBR` 제거
  - 샘플 데이터 추가(`SITE/BIZ/ANALYSIS_ITEM/USER_SITE_ROLE`)

## 4) 다음 구현 우선순위

1. PAS 로그인/JWT + 권한 판정
2. 분석항목 CRUD + 실행기(모듈 경로 실행)
3. SAS 키 검증 + 서비스 API 조회
4. 외부 DB 연동 1종 + 간단 ML 처리 -> `TB_ANALYSIS_RUN` 저장

## 5) 2026-04-24 실제 반영 작업 (인증/로그인 중심)

### 5-1. 인증 구조 반영

- `PAS` 인증 구현 시작:
  - 로그인 API: `POST /api/v1/pas/auth/login`
  - 내 정보 API: `GET /api/v1/pas/auth/me`
- `PAS` 라우트 보호 적용:
  - `auth` 라우트 제외한 PAS 하위 라우트에 JWT 검증 의존성 적용
- `SAS` 키 검증 구조 유지:
  - `X-BIZ-NO`, `X-API-KEY` 기반 검증

### 5-2. 기술 반영 상세

- 설정 확장:
  - `JWT_SECRET_KEY`, `JWT_ALGORITHM`, `JWT_ACCESS_TOKEN_EXP_MIN`
  - 파일: `backend/app/core/config.py`, `backend/.env.example`
- 보안 유틸 확장:
  - 비밀번호 검증(`verify_password`)
  - JWT 생성/검증(`create_access_token`, `decode_access_token`)
  - 파일: `backend/app/core/security.py`
- 사용자 도메인 구현:
  - 로그인 조회/실패횟수증가/성공갱신/사이트권한조회
  - 파일: `backend/app/domain/user/repository.py`, `service.py`, `schema.py`
- 의존성 구조 분리:
  - `dependencies.py` -> `dependencies/` 패키지로 전환
  - `db.py`, `sas_auth.py`, `pas_auth.py`

### 5-3. API 요청 스키마 정책 변경

- 보안 요청으로 로그인 파라미터명 변경:
  - `password` -> `usr_pwd`
  - 파일: `backend/app/domain/user/schema.py`, `backend/app/api/v1/pas/auth.py`

### 5-4. DB 연결/실행 관련 반영

- MariaDB 연결 기준으로 `.env` 반영:
  - `DATABASE_URL=mysql+pymysql://...`
- 드라이버 의존성 반영:
  - `PyMySQL` 추가 (`backend/pyproject.toml`)

### 5-5. 장애/이슈 및 조치

- 이슈:
  - `Access denied for user ...` (DB 계정 host 권한 또는 비밀번호 이슈)
  - 로그인 실패 `비밀번호 불일치`
- 조치:
  - 비밀번호 URL 인코딩(`#` -> `%23`) 적용
  - 로그인 실패 원인 추적 로그 추가(`user 없음/상태비정상/잠금/비밀번호불일치`)
  - `TB_USER.PW_HASH_CN` 실제 argon2 해시로 업데이트 필요 확인
