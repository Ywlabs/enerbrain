# EnerBrain — 제품 맥락

## 왜 존재하는가

고객사별로 제각각인 예측/분석 작업을 중앙 플랫폼에서 통합 실행·관리하기 위한 오케스트레이션 서버가 필요하다.

## 해결하려는 문제

- 프로젝트별 분석 스크립트/주기/결과 관리가 분산되어 운영 난이도 증가
- 고객사 API 제공 시 키 발급/만료/폐기/호출추적 체계 부재
- 운영자가 사이트별 환경을 한 곳에서 관리하기 어려움

## 현재 확정된 사용자 시나리오 (MVP)

1. 사이트(`SITE`) 등록
2. 프로젝트(`BIZ`) 등록
3. 분석항목(`TB_ANALYSIS_ITEM`) 등록
   - 분석명, 알고리즘코드, 크론주기, 파이썬 파일경로, 실행함수명
4. 배치 실행 후 이력(`TB_ANALYSIS_RUN`) 확인
5. 고객사 서비스 API 메타(`TB_API_SVC`) 등록 및 테스트
6. 프로젝트 API 키(`TB_BIZ_API_KEY`) 발급 후 고객사 호출
7. 내부 운영자는 PAS에서 사용자/사이트/프로젝트/분석항목 관리

## 인증/권한 정책

- SAS(고객사 API):
  - 프로젝트 키 필수 (`TB_BIZ_API_KEY`)
- PAS(내부 운영 API):
  - 관리자 JWT 예정
  - 전역권한 + 사이트권한 모델 적용
- 전체관리자:
  - `TB_USER.GLOBAL_ROLE_CD='SUPER_ADMIN'`이면 전체 사이트 접근 가능

## 운영 UX 목표

- 분석항목 중심의 실행현황/실패원인 확인
- 서비스 API 테스트 요청/응답 샘플 관리
- API 요청 로그/감사 로그 기반 운영 추적
