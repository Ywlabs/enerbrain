# EnerBrain — 제품 맥락

## 왜 존재하는가

고객사별 에너지 데이터를 표준화하여 예측 모델로 연결하고, 결과를 API로 제공하는 운영형 분석 플랫폼이 필요하다.

## 해결하려는 문제

- 고객사마다 다른 원천 포맷(DB/API/CSV/SFTP)으로 인해 분석 파이프라인이 반복 개발되는 문제
- 예측 운영에 필요한 데이터 품질/버전/이력 관리 부재
- 고객사 프로젝트별 인증키 발급·운영·교체(로테이션) 체계 부재

## 현재 확정된 사용자 시나리오

1. 고객사(`SITE`) 생성
2. 프로젝트(`BIZ`) 생성 및 PM 배정
3. 수집 방식 선택(DB 연결 또는 SFTP/파일 전달)
4. 매핑 규칙 설정(원천 컬럼 -> 표준 지표 코드)
5. 시계열 적재/품질 점검 후 모델 학습
6. D+1 예측 API 호출(`target_ymd` 기반)
7. 프로젝트 단위 API 키 발급/회수/교체 관리

## 수집 정책

- 기본 원칙: **입구는 유연하게, 분석 레이어는 엄격하게**
- 원천 데이터는 `TB_TS_RAW_JSON`으로 수용 가능
- 분석용 데이터는 `TB_TS_FACT` 표준 구조로 정규화
- 고객사가 DB 연결 불가 시 SFTP/CSV 전달 방식 허용

## 인증/키 정책

- 프로젝트 키: `TB_BIZ_API_KEY` (프로젝트 단위)
- 오픈 API 키: `TB_OPEN_API_KEY` + `TB_OPEN_API_KEY_SVC`
- 키 비교는 해시 기반(`KEY_HASH_CN`), 평문 저장 금지

## 운영 UX 목표

- 매핑 버전 관리 + 품질 결과 시각화
- API 요청 로그/감사 로그 기반 운영 추적
- 관리자 화면에서 프로젝트 키 발급/폐기/교체 지원
